# G1 MuJoCo 站立仿真节点接口 I/O 文档

## 1. 接口范围

- 模块名：G1 MuJoCo 站立仿真节点
- 仿真循环频率：$f_{sim}=1/\text{SIMULATE\_DT}$
- 可视化循环频率：$f_{viewer}=1/\text{VIEWER\_DT}$

## 2. 输入 I（Inputs）

| 字段名 | 类型/维度 | 单位 | 更新频率 | 来源 | 去向 |
| --- | --- | --- | --- | --- | --- |
| config.ROBOT_SCENE | str | - | 启动一次 | 配置文件 | MjModel.from_xml_path |
| config.SIMULATE_DT | float | s | 启动一次 | 配置文件 | mj_model.opt.timestep |
| config.VIEWER_DT | float | s | 启动一次 | 配置文件 | PhysicsViewerThread 的 sleep |
| config.DOMAIN_ID | int | - | 启动一次 | 配置文件 | ChannelFactoryInitialize |
| config.INTERFACE | str | - | 启动一次 | 配置文件 | ChannelFactoryInitialize |
| config.ENABLE_ELASTIC_BAND | bool | - | 启动一次/运行时判断 | 配置文件 | 决定是否挂载外力与键盘回调 |
| config.ROBOT | str | - | 启动一次 | 配置文件 | 选择 torso_link/base_link 施力目标 |
| config.PRINT_SCENE_INFORMATION | bool | - | 启动一次 | 配置文件 | 控制是否打印场景信息 |
| mj_data.qpos（关节部分） | float[N] | rad（转动关节） | $f_{sim}$ | MuJoCo 状态 | 控制器 q 反馈 |
| mj_data.qvel（关节部分） | float[N] | rad/s | $f_{sim}$ | MuJoCo 状态 | 控制器 dq 反馈 |
| mj_data.xmat（torso_link） | float[3x3] | 无量纲 | $f_{sim}$ | MuJoCo 状态 | 计算躯干 roll/pitch |
| mj_data.qvel[3:6]（基座角速度） | float[3] | rad/s | $f_{sim}$ | MuJoCo 状态 | 姿态阻尼项 |
| elastic_band.enable | bool | - | 事件驱动 | 键盘回调 | 决定是否施加外力 |
| elastic_band.Advance(qpos[:3], qvel[:3]) 输入 | float[3], float[3] | m, m/s | $f_{sim}$（启用时） | MuJoCo 状态 | 计算外力 |

说明：N 为受控关节数（当前 JOINT_NAMES 为 29）。

## 3. 输出 O（Outputs）

| 字段名 | 类型/维度 | 单位 | 更新频率 | 来源 | 去向 |
| --- | --- | --- | --- | --- | --- |
| mj_data.ctrl[:nu] | float[nu] | 通常为 N·m（取决于执行器定义） | $f_{sim}$ | G1StandController.apply() | MuJoCo 执行器输入 |
| mj_data.xfrc_applied[link,:3] | float[3] | N | $f_{sim}$（启用时） | ElasticBand.Advance | 对机体施加外力 |
| mj_data（全状态） | 结构体 | 混合 | $f_{sim}$ | mujoco.mj_step | 下一周期反馈、可视化、桥接读取 |
| Viewer 图像帧 | 渲染帧 | - | $f_{viewer}$ | viewer.sync() | 屏幕显示 |
| 场景打印信息 | 文本 | - | 启动一次（可选） | unitree.PrintSceneInformation() | 控制台输出 |

## 4. 内部控制接口（控制器子模块）

| 字段名 | 角色 | 单位 | 更新频率 | 来源 | 去向 |
| --- | --- | --- | --- | --- | --- |
| neutral_pose | 初始参考位姿 | rad | 初始化一次 | qpos 启动快照 | 目标插值起点 |
| stand_pose | 站立目标位姿 | rad | 初始化一次 | neutral_pose + stance_offsets | 目标插值终点 |
| blend = min(1, elapsed/ramp_time) | 软启动插值系数 | - | $f_{sim}$ | 时间戳 | 混合 neutral_pose/stand_pose |
| roll_correction/pitch_correction | 姿态补偿量 | rad 等效量 | $f_{sim}$ | xmat, qvel[3:6] | 分配到腰/髋/踝目标 |
| torque | 关节扭矩命令 | N·m（常见） | $f_{sim}$ | PD + 限幅 | 写入 mj_data.ctrl |

## 5. 端到端数据流（简版）

1. 配置与模型加载：config + XML -> mj_model/mj_data
2. 状态采样：qpos/qvel/xmat -> G1StandController.apply()
3. 控制输出：torque -> mj_data.ctrl
4. 动力学推进：mj_step 更新 mj_data
5. 可选外力：ElasticBand -> xfrc_applied
6. 可视化：viewer.sync() 输出画面
